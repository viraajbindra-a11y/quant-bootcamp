def daily_returns(prices):

Day 4 — Probability and Expected Value

Math Finance Python

Time invested: ~3 hours. Goal: Use probability to predict outcomes.

Expected Value Concept

Expected value is the average outcome if an event happens many times.

Formula: E(X) = (outcome1 × probability1) + (outcome2 × probability2) + ...

stock_up_gain = 100
stock_up_prob = 0.6
stock_down_loss = 40
stock_down_prob = 0.4

expected_value = (stock_up_gain * stock_up_prob) + (-stock_down_loss * stock_down_prob)
print(expected_value)
44.0

On average, this stock investment returns $44.

Probability Distributions

Stock returns follow a bell curve (normal distribution). Most days are near the average; extreme days are rare.

import random
returns = [random.gauss(0.001, 0.02) for _ in range(1000)]
print(f"Average return: {sum(returns)/len(returns)}")
print(f"Min: {min(returns)}, Max: {max(returns)}")
Average return: 0.00098 Min: -0.0612, Max: 0.0743

Day 5 — Standard Deviation and Risk Measurement

Math Finance Python

Time invested: ~3 hours. Goal: Measure how much stock prices vary.

Standard Deviation Formula

Std Dev: How far, on average, values spread from the mean.

Formula: σ = √( Σ(x - μ)² / n )

import math

def std_dev(numbers):
    mean = sum(numbers) / len(numbers)
    variance = sum((x - mean)**2 for x in numbers) / len(numbers)
    return math.sqrt(variance)

returns = [0.001, -0.005, 0.008, -0.002, 0.006]
sd = std_dev(returns)
print(f"Std Dev: {sd}")
Std Dev: 0.00503

Z-Score and Tail Risk

Z-score = (value - mean) / std_dev. Tells you how unusual a value is.

return_value = -0.05
mean_return = 0.001
std_dev_return = 0.02

z_score = (return_value - mean_return) / std_dev_return
print(f"Z-Score: {z_score}")
Z-Score: -2.55

A z-score of -2.55 means this return is 2.55 standard deviations below average — unusual but possible.

The 68-95-99.7 Rule

In a normal distribution: 68% of data within 1 std dev, 95% within 2, 99.7% within 3.

Day 6 — The Sharpe Ratio and Risk-Adjusted Returns

Math Finance Python

Time invested: ~3 hours. Goal: Compare investments by risk, not just returns.

Why Sharpe Ratio Matters

Investment A: 20% return, 30% volatility. Investment B: 15% return, 10% volatility. Which is better? Sharpe Ratio answers this.

Formula: Sharpe Ratio = (Return - Risk-Free Rate) / Std Dev

Sharpe Ratio Calculator

def sharpe_ratio(returns, risk_free_rate=0.02):
    avg_return = sum(returns) / len(returns)
    std_dev = calculate_std_dev(returns)
    annual_return = avg_return * 252  # 252 trading days/year
    annual_std = std_dev * math.sqrt(252)
    return (annual_return - risk_free_rate) / annual_std

stock_returns = [0.001, -0.005, 0.008, 0.003, 0.006]
sharpe = sharpe_ratio(stock_returns)
print(f"Sharpe Ratio: {sharpe}")
Sharpe Ratio: 1.42

A Sharpe Ratio > 1.0 is good. This investment is generating 1.42 units of return per unit of risk taken.

Value at Risk (VaR)

VaR answers: "What's the worst daily loss I should expect 95% of the time?"

If VaR = -2.37%, then on 95% of days, losses won't exceed 2.37%. Banks must calculate and report VaR daily.

🚀 Welcome to Trading Algorithms

Up until now, we've analyzed static data. Today, we start building tools that traders actually use: moving averages and trend analysis.

What is a Moving Average?

A moving average smooths out price noise to reveal the underlying trend. It's the average of the last N prices (the "window").

Simple Moving Average (SMA): Average of the last N closing prices.

prices = [100, 102, 101, 103, 105, 104, 106, 108, 107, 109]

def moving_average(prices, window):
    mas = []
    for i in range(window - 1, len(prices)):
        ma = sum(prices[i-window+1:i+1]) / window
        mas.append(ma)
    return mas

ma_5 = moving_average(prices, 5)
print(ma_5)
[102.2, 103.2, 104.2, 106.0, 106.8]

Each value is the average of the last 5 prices. A 5-day moving average smooths daily noise.

Trading Strategies with Moving Averages

Crossover Strategy: Buy when price crosses above the MA (momentum), sell when it crosses below.

SMA Crossover Logic:

  • If price > MA, trend is up. Consider buying.
  • If price < MA, trend is down. Consider selling.
  • The longer the window, the smoother the signal (fewer false trades, slower response).

Backtesting a Strategy

Backtesting = testing a strategy on historical data to see if it would have worked.

def test_strategy(prices, window_size):
    mas = moving_average(prices, window_size)
    trades = []
    position = None  # None = no stock, "long" = holding

    for i in range(window_size - 1, len(prices)):
        price = prices[i]
        ma = mas[i - (window_size - 1)]

        if price > ma and position != "long":
            trades.append(("buy", price))
            position = "long"
        elif price < ma and position == "long":
            trades.append(("sell", price))
            position = None

    return trades

trades = test_strategy(prices, 5)
for action, price in trades:
    print(f"{action} @ ${price}")
buy @ $105 sell @ $103 buy @ $106 ... (continues)

Crossover Strategy

Window Size Tradeoffs

Backtesting
Testing a strategy on past data. Shows if it would have made money historically.
