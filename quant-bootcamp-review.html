Python Finance

Step 1: Lists — Storing Multiple Values

A list holds multiple values in order. Learned to create lists, access items by index, and use built-in functions.

prices = [150, 152, 148, 155, 160, 157, 163]

print(prices)
print(prices[0])      # First item (Python counts from 0!)
print(prices[3])      # Fourth item
print(len(prices))    # How many items

prices.append(170)    # Add to the end
print("Highest:", max(prices))
print("Lowest:", min(prices))
print("Total:", sum(prices))
print("Average:", sum(prices) / len(prices))
Key Concept — Zero-Based Indexing

Python starts counting at 0, not 1. So prices[0] is the first item and prices[3] is actually the fourth. This is universal across almost all programming languages.

Step 2: For Loops — Processing Every Item

A loop runs the same code on every item in a list. Learned three patterns:

Pattern 1: Loop through and print

for price in prices:
    print(price)

Pattern 2: Loop with conditions (combining Day 1 skills)

for price in prices:
    if price > 155:
        print(str(price) + " - above 155")
    else:
        print(str(price) + " - at or below 155")

Pattern 3: Loop with counting

above = 0
for price in prices:
    if price > 155:
        above = above + 1
print(str(above) + " out of " + str(len(prices)) + " days")
Why This Matters for Quant Finance

Real financial data has thousands of data points. You can't manually check each one. Loops let you process 10,000 stock prices the same way you'd process 10. The code doesn't change — only the data does.

Day 3 — Functions

Python Finance

Time invested: ~1 hour (in progress). Goal: Package code into reusable blocks — the way real software is organized.

Step 1: Understanding def, Parameters, and return

A function is like saving a formula on a calculator. You define it once, then call it with different inputs forever.

def calculate_return(buy, sell):
    return (sell - buy) / buy

Breaking it down:

def
Tells Python "I'm defining a new function"
calculate_return
The name — you pick it, it should describe what it does
(buy, sell)
Parameters — placeholders that get filled in when you call it
return
Sends the answer back to wherever the function was called

Testing with different inputs:

print(calculate_return(100, 115))   # 0.15 (15% gain)
print(calculate_return(50, 75))     # 0.5 (50% gain)
print(calculate_return(200, 180))   # -0.1 (10% loss)
0.15 0.5 -0.1
Learning Moment — Mental Math Check

Predicted calculate_return(50, 75) would give 0.2 (20%), but Python returned 0.5 (50%). Walking through the math: (75-50)/50 = 25/50 = 0.5. The error was in the mental math, not Python. $25 profit on a $50 investment is half your money back = 50%. Always divide by the buy price because that's what you invested.

Insight

When your mental math disagrees with Python, walk through the formula step by step. The computer doesn't make arithmetic mistakes — but it will blindly execute a wrong formula if you write one.

Step 2: Average Function

def average(numbers):
    return sum(numbers) / len(numbers)

prices = [150, 152, 148, 155, 160]
print(average(prices))
153.0

Verified the answer mentally: (150+152+148+155+160) = 765, divided by 5 = 153. Python and mental math matched.

Key Insight

One function works on ANY list. Pass it stock prices, returns, temperatures, test scores — it doesn't care. Write it once, use it forever. This is why functions are the building blocks of all software.

Step 3: Daily Returns Function

Wrapped the Day 2 return calculator into a reusable function.

def daily_returns(prices):
    returns = []
    for i in range(1, len(prices)):
        r = (prices[i] - prices[i - 1]) / prices[i - 1]
        returns.append(r)
    return returns
Learning Moment — Indentation Error

First attempt gave: SyntaxError: 'return' outside function. The return returns line wasn't indented inside the function. In Python, indentation isn't cosmetic — it's how the language knows which lines belong to which block.

SyntaxError: 'return' outside function

After fixing indentation (lining up return with the for loop, one level inside the function), it worked:

[0.013333333333333334, -0.02631578947368421, 0.0472972972972973, 0.03225806451612903]

Those are the daily returns for prices [150, 152, 148, 155, 160]:

DayPrice ChangeReturnMeaning
1→2150 → 152+1.3%Up day
2→3152 → 148-2.6%Down day
3→4148 → 155+4.7%Up day
4→5155 → 160+3.2%Up day
Why Decimals, Not Percentages?

Returns are stored as decimals (0.013 instead of 1.3%) because the math is cleaner. If a stock goes up 5% then down 5%, you multiply: 1.05 × 0.95 = 0.9975 — meaning you're actually down 0.25%. With percentages you'd need extra conversion steps.

Step 4: Functions Working Together

Milestone Combined multiple functions — calling daily_returns() and passing the result to average().

prices = [150, 152, 148, 155, 160]
results = daily_returns(prices)
print("Average daily return: " + str(round(average(results) * 100, 2)) + "%")
print("Best day: " + str(round(max(results) * 100, 2)) + "%")
print("Worst day: " + str(round(min(results) * 100, 2)) + "%")
The Lego Principle

Small functions snap together like legos. daily_returns() produces data, average() analyzes it. Need to do this for 500 stocks? Call the same two functions 500 times. One function, infinite uses. This is how quant teams build their toolkits.

Today's Plan

Part 1 — Probability Basics in Python (30 min)

Since the math background is strong (Algebra 2), this part moves fast. Focus is on coding probability calculations, not learning the theory from scratch.

Key concepts being covered:

Independent Events
P(A and B) = P(A) × P(B). Like a stock going up 5 days in a row: 0.6⁵ = 7.78%
Streaks Are Rare
Even with 60% daily odds, 5 up days in a row is only ~8%. Quants never assume streaks.

First program of the day:

p_up = 0.60
p_five_up = p_up ** 5
print("Probability of 5 up days in a row: " + str(round(p_five_up * 100, 2)) + "%")
Quant Insight

Even when odds are in your favor, consecutive wins are surprisingly unlikely. This is why quants think in probabilities and large sample sizes, not individual bets. One trade doesn't matter — a thousand trades reveal the edge.

Part 2 — Coin Flip Simulator (30 min)

Introduction to the random module — Python's randomness engine. Build a simulator that flips a coin thousands of times and checks if the simulated results match the mathematical prediction.

New Python concepts: import random, random.choice(), simulation loops

Key idea: When you can't solve a probability problem with math alone, you can simulate it with code and get a very close answer.

Part 3 — Monte Carlo Stock Simulation (1 hour)

The big one. Monte Carlo simulation is one of the most important techniques in quantitative finance.

What Is Monte Carlo?
Run a random simulation thousands of times to estimate probabilities that are too complex to calculate by hand.
The Finance Version
If a stock goes up 1% with 55% probability and down 1% with 45% probability each day, simulate 252 trading days (one year) thousands of times.
What You'll Answer
What percentage of the time do you make money? What's the average outcome? What's the worst case?
Why Quants Use This
Real markets are too complex for simple formulas. Monte Carlo lets you model uncertainty and make decisions under randomness.
Why This Day Matters

After today, you'll be able to take any "what are the odds?" question and answer it two ways: with math (when possible) and with simulation (always). That's a genuine quant skill. Hedge funds use Monte Carlo simulations every single day to manage billions of dollars in risk.

Probability

Streak probability: Calculated that a stock with 60% daily up probability has only a 7.78% chance of going up 5 days in a row (0.6⁵). Initial guess was 3% — probability intuition is often wrong, which is exactly why quants use math.

Simulation

Coin flip simulator: Used import random and random.choice() to simulate 1,000 coin flips (506 heads = 50.6%). Then 100,000 flips (49,876 = 49.876%). Demonstrated the Law of Large Numbers — more trials = closer to true probability.

Monte Carlo

Monte Carlo stock simulator: Built a full simulation — 10,000 possible futures of a stock over 252 trading days. A 55% daily edge with 1% moves gave a 93.56% win rate. Calculated best case ($2,287), worst case ($717), average ($1,287), and Value at Risk at the 1% level ($875).

Discovery

Ran 4 experiments comparing edge size and move size:

SetupWin RateBestWorstInsight
55% edge, 1% moves93.56%$2,287$717Small edge + small moves = very reliable
50% (no edge), 1% moves48.13%$1,694$564No edge = coin flip, as expected
55% edge, 5% moves87.74%$2,380$702Edge helps but big moves add risk
50% (no edge), 5% moves32.83%$12,024$44Volatility drag destroys returns
Key Discovery — Volatility Drag

With 50/50 odds and 5% daily moves, win rate dropped to just 32.83% — far below the expected 50%. This happens because percentage losses are harder to recover from (lose 5% from $1,000 = $950, but need 5.26% to get back). The median ($729) was far below the average ($997), meaning a few lucky outliers pulled the average up while most simulations lost money. Best case was $12,024 but worst was $44 — extreme range.

Quant Lesson

A small consistent edge with small moves beats a big volatile game every time. This is why real quant strategies make thousands of small trades, not a few big bets. The house always wins — not by winning big, but by winning slightly, repeatedly.

Key Rules to Remember

1. Python counts from 0. list[0] is the first item, not list[1].

2. Indentation matters. Code inside functions, loops, and if-statements MUST be indented. This isn't cosmetic — it's how Python knows which code belongs where.

3. Returns are stored as decimals. 0.15 means 15%. Multiply by 100 only when displaying to humans.

4. range(a, b) excludes b. range(1, 5) gives 1, 2, 3, 4 — not 5.

5. return ≠ print(). return sends data back silently. print() shows it on screen. A function usually returns; the caller decides whether to print.
