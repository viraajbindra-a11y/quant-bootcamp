Part 3 — Monte Carlo Stock Simulation (1 hour)

The big one. Monte Carlo simulation is one of the most important techniques in quantitative finance.

What Is Monte Carlo?
Run a random simulation thousands of times to estimate probabilities that are too complex to calculate by hand.
The Finance Version
If a stock goes up 1% with 55% probability and down 1% with 45% probability each day, simulate 252 trading days (one year) thousands of times.
What You'll Answer
What percentage of the time do you make money? What's the average outcome? What's the worst case?
Why Quants Use This
Real markets are too complex for simple formulas. Monte Carlo lets you model uncertainty and make decisions under randomness.
Why This Day Matters

After today, you'll be able to take any "what are the odds?" question and answer it two ways: with math (when possible) and with simulation (always). That's a genuine quant skill. Hedge funds use Monte Carlo simulations every single day to manage billions of dollars in risk.

Day 5 — Statistics: Mean, Variance, Standard Deviation & Sharpe Ratio

Statistics Risk Analysis Functions

The day we learned how quants actually measure risk. Not "this stock feels risky" — actual numbers that tell you exactly how much a stock bounces around and whether the reward justifies the ride.

Part 1: Mean — The Center of Your Data

Started with a list of 5 daily stock returns and calculated the mean (average) by hand using code — sum() divided by len().

returns = [0.02, -0.01, 0.03, -0.005, 0.015]
total = sum(returns)
count = len(returns)
mean = total / count
print("Mean daily return:", mean)
print("That's", round(mean * 100, 2), "% per day")
Mean daily return: 0.01 That's 1.0 % per day

Quant insight: If a stock has a mean daily return of 1%, you'd expect it to go up about 1% on any given day. But "expect" doesn't mean "guarantee."

Part 2: Variance & Standard Deviation — Measuring Risk

The key question: how much does the stock bounce around that mean? Built the full calculation step by step:

# Step 1: Differences from mean
differences = [r - mean for r in returns]
# Step 2: Square each difference
squared_diffs = [d ** 2 for d in differences]
# Step 3: Variance = average of squared differences
variance = sum(squared_diffs) / len(squared_diffs)
# Step 4: Standard deviation = square root of variance
std_dev = variance ** 0.5
Variance: 0.00023 Standard deviation: 0.0152 That means returns typically vary by 1.52% from the mean

Key lesson: Why square the differences? Without squaring, positives and negatives cancel out and you get zero — which tells you nothing. Squaring makes everything positive, then the square root brings it back to the original scale.

Mental model: Mean ± 1 std dev = the range where ~68% of days land. For this stock: between -0.52% and +2.52%.

Part 3: Comparing Stocks — The Sharpe Ratio

Compared two stocks that both average 1% daily return:

stock_a = [0.01, 0.01, 0.01, 0.01, 0.01]   # steady
stock_b = [0.10, -0.08, 0.15, -0.12, 0.00]  # wild

Built a calc_stats() function that computes mean, std dev, and a simplified Sharpe Ratio (mean ÷ std dev):

--- Stock A (Steady) --- Mean return: 1.0% Std deviation: 0.0% Risk/Reward ratio: inf Verdict: GREAT - reward beats the risk --- Stock B (Wild) --- Mean return: 1.0% Std deviation: 10.28% Risk/Reward ratio: 0.1 Verdict: BAD - too much risk for the reward

Breakthrough concept: The Sharpe Ratio = reward per unit of risk. Every hedge fund in the world obsesses over this number. Stock A has infinite Sharpe (no risk, all reward). Stock B has 0.1 (terrible — 10% swings for 1% expected gain).

The mean alone never tells you the full story — you always need the standard deviation too.

Day 6 — Normal Distribution, Z-Scores & Value at Risk

Statistics Risk Analysis Simulation

The day we learned the bell curve — the mathematical shape that stock returns actually follow — and used it to predict the probability of crashes, calculate Value at Risk, and verify theory against simulation.

Part 1: The 68-95-99.7 Rule

Generated 100,000 random returns using random.gauss() and verified one of the most important rules in statistics:

68% Rule
~68% of data falls within 1 standard deviation of the mean
95% Rule
~95% of data falls within 2 standard deviations
99.7% Rule
~99.7% of data falls within 3 standard deviations
Why It Matters
Tells you how rare extreme events are — a 3-sigma move happens only 0.3% of the time
mean = 0.001
std = 0.02
data = [random.gauss(mean, std) for _ in range(100000)]

within_1 = sum(1 for x in data if mean-std <= x <= mean+std)
within_2 = sum(1 for x in data if mean-2*std <= x <= mean+2*std)
within_3 = sum(1 for x in data if mean-3*std <= x <= mean+3*std)
Within 1 std dev: 68.17% (theory: 68.27%) Within 2 std devs: 95.39% (theory: 95.45%) Within 3 std devs: 99.73% (theory: 99.73%)
Key Insight

The simulation matched the theory almost exactly — confirming that random.gauss() truly generates bell-curve distributed data. This is the Law of Large Numbers in action: 100,000 data points is enough for the pattern to emerge.

Part 2: Z-Scores — How Extreme Is This Day?

A z-score converts any return into "how many standard deviations from normal is this?" Formula: z = (return - mean) / std

def z_score(ret, mean, std):
    return (ret - mean) / std

# Black Monday (Oct 19, 1987): -22.6% drop
z = z_score(-0.226, 0.001, 0.02)
# z = -11.35 → "11 standard deviations below normal"

Z-score table for tested days:

EventReturnZ-ScoreMeaning
Normal up day+1.5%+0.70Totally normal
Good day+3%+1.45Above average but not rare
Bad day-2%-1.05Within 1 std dev — happens often
Flash crash-5%-2.55Beyond 2 sigma — rare (~1% chance)
Black Monday-22.6%-11.35Should "never" happen — but it did
Why Black Monday Matters

A z-score of -11.35 is so extreme that the normal distribution says it should happen once every 1028 years. But it happened. This means stock returns have "fat tails" — extreme events are more common than the bell curve predicts. Every quant knows this.

Part 3: Risk Calculator with normal_cdf

Built a function to approximate the cumulative normal distribution (using the Abramowitz and Stegun method) — calculating the exact probability that a return falls below any threshold.

def normal_cdf(z):
    if z < -8: return 0.0
    if z > 8: return 1.0
    t = 1 / (1 + 0.2316419 * abs(z))
    d = 0.3989422804 * math.exp(-z * z / 2)
    p = d * t * (0.3193815 + t * (-0.3565638 + t * (1.781478 +
        t * (-1.821256 + t * 1.330274))))
    if z > 0: return 1 - p
    return p

Used this to build a risk_report() function that answers: what's the probability of loss? Of a 5% crash? What's the Value at Risk?

--- Risk Report for SAFE_STOCK (mean=0.1%, std=1.5%) --- Probability of ANY loss: 47.3% Probability of 5% crash: 0.04% Value at Risk (95%): -2.37% Expected daily range: -2.87% to 3.07%
Value at Risk (VaR)

VaR at 95% confidence answers: "What's the worst daily return I should expect 95% of the time?" If VaR is -2.37%, that means on 95% of days, losses won't exceed 2.37%. The other 5%? That's where the risk lives. Banks are legally required to calculate and report their VaR.

Part 4: Theory vs. Simulation — Do They Match?

The ultimate test: does the mathematical formula agree with running 100,000 simulated days?

--- Comparison: VOLATILE_STOCK --- Metric | Theory | Simulation | Match? P(loss) | 48.0% | 47.7% | MATCH P(5% crash) | 2.28% | 2.26% | MATCH VaR 95% | -6.23% | -6.18% | MATCH Verdict: Theory and simulation AGREE

Key Rules to Remember

1. Python counts from 0. list[0] is the first item, not list[1].

2. Indentation matters. Code inside functions, loops, and if-statements MUST be indented. This isn't cosmetic — it's how Python knows which code belongs where.

3. Returns are stored as decimals. 0.15 means 15%. Multiply by 100 only when displaying to humans.

4. range(a, b) excludes b. range(1, 5) gives 1, 2, 3, 4 — not 5.

5. return ≠ print(). return sends data back silently. print() shows it on screen. A function usually returns; the caller decides whether to print.
